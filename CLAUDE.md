### Milestone 1.2 — Bayesian PMF in PyMC

**File:** `scripts/models/bayesian_pmf.py`

Implement the model strictly following this specification:

```python
import pymc as pm
import pytensor.tensor as pt
import numpy as np
import arviz as az
from typing import Optional

class BayesianPMF:
    """
    Bayesian Probabilistic Matrix Factorization.

    Model:
        U[u] ~ Normal(0, sigma_u)   shape: (n_users, n_factors)
        V[i] ~ Normal(0, sigma_v)   shape: (n_items, n_factors)
        sigma_u ~ HalfNormal(1)
        sigma_v ~ HalfNormal(1)
        tau     ~ HalfNormal(1)     (observation noise precision)
        R[u,i]  ~ Normal(U[u] @ V[i].T, 1/tau)  for observed (u,i)

    Priors are mildly regularising; avoid Uniform priors.
    """

    def __init__(self, n_factors: int = 10, random_seed: int = 42):
        ...

    def build_model(
        self,
        ratings: np.ndarray,       # (n_users, n_items), 0 = unobserved
        mask: np.ndarray,           # boolean, True = observed
    ) -> pm.Model:
        """Return a compiled PyMC model. Store as self.model."""
        ...

    def fit(
        self,
        draws: int = 500,
        tune: int = 500,
        chains: int = 2,
        target_accept: float = 0.9,
    ) -> az.InferenceData:
        """Run NUTS. Store trace as self.trace. Return InferenceData."""
        ...

    def predict(self, user_idx: int, item_idx: int) -> dict:
        """
        Return dict with:
          'mean': posterior mean rating
          'std':  posterior std (epistemic uncertainty)
          'hdi_94': 94% HDI interval
        """
        ...

    def recommend_topk(self, user_idx: int, k: int = 10) -> list[int]:
        """Return top-k item indices by posterior mean rating for user."""
        ...
```

class BayesianPropensityModel:
    """
    Models P(O_{ui} = 1 | user u, item i) — the probability an item is
    observed/exposed to a user.

    Model:
        logit(p_{ui}) = alpha_u + beta_i
        alpha_u ~ Normal(mu_alpha, sigma_alpha)  # user intercept

```python
class DeconfoundedRecommender:
    """
    Two-stage causal debiasing:

    Stage 1 — Factor Model (substitute confounder):
        Fit a probabilistic factor model on the ITEM exposure matrix to
        extract a latent factor Z per user that serves as a substitute
        for the unmeasured confounder (e.g. 'mood', 'session context').
        Use Bayesian PMF on the binary observation mask.

    Stage 2 — Debiased Outcome Model:
        Fit a rating prediction model that conditions on Z (the substitute
        confounder), implementing approximate backdoor adjustment.
        P(R_{ui} | do(A_{ui}=a)) ≈ E_Z [ P(R_{ui} | A_{ui}=a, Z) ]

    Important caveat: include a clear docstring and notebook note that
    identifiability of the substitute confounder approach has been
    critiqued (Ogburn et al.); honest evaluation on randomised test
    sets (Coat/Yahoo) is essential to avoid over-claiming causal
    identification.
    """

    def fit_factor_model(self, observation_mask: np.ndarray) -> np.ndarray:
        """Stage 1: returns substitute confounder Z (n_users, n_factors)."""
        ...

    def fit_outcome_model(
        self,
        ratings: np.ndarray,
        mask: np.ndarray,
        Z: np.ndarray,
    ) -> az.InferenceData:
        """Stage 2: outcome model conditioned on Z."""
        ...

    def counterfactual_relevance(
        self,
        user_idx: int,
        item_indices: list[int],
    ) -> np.ndarray:
        """
        Use PyMC's do-operator to compute:
          E[R_{ui} | do(A_{ui} = 1)]
        i.e. the interventional expected rating if the item WERE exposed,
        regardless of the natural exposure mechanism.

        Returns array of (mean, std) tuples — posterior mean and uncertainty
        under the interventional distribution.
        """
        ...
```

class BayesianThompsonSampler:
    """
    Thompson Sampling recommender using posterior samples from BayesianPMF.

    At each recommendation step for user u:

class FeedbackLoopSimulator:
    """
    Simulates a sequential recommendation feedback loop over T rounds.

    At each round t:
      1. Recommender selects items for each user (Thompson / Greedy / Random)
      2. Users 'interact' with items according to true relevance
         (use Coat/Yahoo unbiased ratings as ground truth)
      3. New observations are added to training data
      4. Model is re-fit (or updated via approximate update)
      5. Track: popularity distribution of recommended items,
                coverage (fraction of item catalogue recommended),
                cumulative NDCG on held-out unbiased test

    The key hypothesis to test:
      Greedy recommendations → popularity bias amplifies each round
      (popular items get recommended → more data → model more confident
      in popular items → recommended even more)

      Thompson Sampling → uncertainty on long-tail items drives
      exploration → popularity distribution stays broader → better
      long-term coverage and unbiased NDCG

    Use T = 10 rounds; refit a lightweight model (or use SVI approximation
    in NumPyro for speed) at each round. If full MCMC per round is too
    slow (>10 min per round), switch to SVI for the simulation loop and
    note this trade-off explicitly.
    """

    def __init__(
        self,
        true_ratings: np.ndarray,    # unbiased ground truth
        n_rounds: int = 10,
        random_seed: int = 42,
    ): ...

    def run(self, strategy: str = "thompson") -> dict:
        """
        Returns dict with per-round metrics:
          'ndcg': list of NDCG@10 per round
          'coverage': list of catalogue coverage per round
          'gini': list of Gini coefficient of recommendation distribution
        """

    """
    NumPyro implementation of Bayesian PMF using Stochastic Variational

## General Implementation Standards

These apply across all phases:

**Code quality:**

- Type hints on all function signatures
- Docstrings: one-line summary + Args + Returns
- No magic numbers — all hyperparameters as named constants or function args
- `random_seed` plumbed through everywhere

**PyMC best practices:**

- Always check R-hat < 1.05 and ESS > 400 after sampling
- Use `pm.Data` containers for arrays that vary between prior/posterior/predictive
- Use `az.summary()` to inspect convergence; log results to a markdown cell
- Save traces with `az.to_netcdf()` to `outputs/traces/`

**Notebook standards:**

- First cell: imports + seed
- Second cell (markdown): what this notebook does and why
- Every plot: title, axis labels, units, legend
- Every results table: column names, units, n_samples used

**Error handling:**

- If MCMC diverges: report it clearly, try increasing `target_accept` to 0.95,
  re-parameterise (non-centered), and document what you tried
- If a dataset download fails: fall back to synthetic, print a clear message
- If a model runs > 15 min on CPU: switch to a smaller subset and note it
